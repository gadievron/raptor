# SYNTEX Nemesis — MITRE ATT&CK Mutations

**2,500 novel attack vectors mapped to MITRE ATT&CK techniques, autonomously generated using reinforcement learning.**

Generated by [SYNTEX Nemesis](https://syntexsecurity.com) — a bounded entropy engine that systematically explores MITRE ATT&CK technique space, generates novel mutations, and validates them against a production RL defense system. No LLMs, no manual curation.

## Files

| File | Vectors | Format |
|------|---------|--------|
| `nemesis_batch1_1000_vectors.jsonl` | 1,000 | JSONL |
| `nemesis_batch1_stix2.1.json` | 1,000 | STIX 2.1 Bundle |
| `nemesis_batch2_1500_vectors.jsonl` | 1,500 | JSONL |
| `nemesis_batch2_stix2.1.json` | 1,500 | STIX 2.1 Bundle |

~900 unique MITRE techniques covered across both batches.

## Vector Schema (JSONL)

```json
{
  "syn_code": "SYN-19540",
  "mitre_tcode": "T1573.001",
  "attack_type": "t1573_001_symmetric_cryptography",
  "payload_signature": "stunnel | powershell.exe | /tmp/*.pem | TLS to non-standard port",
  "detection": {
    "processes": ["stunnel", "powershell.exe", "openssl"],
    "file_artifacts": ["/tmp/*.pem"],
    "network_patterns": ["TLS to non-standard port", "encrypted C2 channel"],
    "cli_patterns": ["openssl s_client", "stunnel config"]
  },
  "keywords": ["stunnel", "powershell.exe", "openssl"],
  "source": "SYNTEX ART (Adversarial Red Team)",
  "license": "CC-BY-4.0"
}
```

Each vector includes:
- **MITRE T-code mapping** for technique identification
- **Payload signature** describing the attack pattern
- **Detection guidance** (processes, file artifacts, network patterns, CLI patterns)
- **Keywords** for search and correlation
